API supported by React Context

React.createContext()

const {Provider, Consumer} = React.createContext(defaultValue);

Creates a { Provider, Consumer } pair. When React renders a context Consumer, it will read the current context value from the closest matching Provider above it in the tree. Meaning, you can have multiple Providers located throughout your application, nested.

Provider

<Provider value={/* some value */}>

A React component that allows Consumers to subscribe to context changes.

The value which is passed to the provider value={/**/} can be accessed by any of the descendants of the enclosing Provider

Providers can also be nested to simplify overriding values deeper within the tree

Consumer

<Consumer>
  {value => /* render something based on the context value */}
</Consumer>
  • A React component that subscribes to context changes
  • Requires a function as child - which is the current context value from the closest Provider
  • On no Provider found for the node, whatever was defined in the defaultValue will be provided
  • üí° Propagation of values from the Provider to any of the Consumer is not subject to the shouldComponentUpdate()-method
  • üí° Internally changes determined by Object.is()

When to use?

  • Need to passing props more than three times
  • Need to passing props more than n times
  • Application not big enough
